# Dokumentasi Firmware ESP8266

## 6.1 Gambaran Umum

Firmware ESP8266 berjalan di mikrokontroler **ESP8266** dan berfungsi sebagai **Pemindai RFID Master**. Membaca kartu RFID, berkomunikasi dengan server backend, dan mengontrol ESP32-CAM melalui koneksi serial.

**Komponen Hardware:**
- **MCU:** ESP8266
- **RFID Reader:** MFRC522 (interface SPI)
- **Display:** LCD 16x2 dengan interface I2C (address 0x27)
- **Buzzer:** Buzzer aktif di GPIO0 (active-low)
- **Serial:** SoftwareSerial untuk komunikasi ESP32-CAM

**Pemetaan Pin:**

| Komponen | Pin | GPIO |
|----------|-----|------|
| MFRC522 SDA(SS) | D8 | GPIO15 |
| MFRC522 RST | D4 | GPIO2 |
| MFRC522 MOSI | D7 | GPIO13 |
| MFRC522 MISO | D6 | GPIO12 |
| MFRC522 SCK | D5 | GPIO14 |
| LCD I2C SDA | D2 | GPIO4 |
| LCD I2C SCL | D1 | GPIO5 |
| Buzzer | D3 | GPIO0 |
| SoftwareSerial RX | RX | GPIO3 |
| SoftwareSerial TX | D0 | GPIO16 |

### 6.2.4 Fungsi

#### `void beep(int durationMs)`

**Tujuan:** Mengaktifkan buzzer selama durasi tertentu.

**Input:** `durationMs` — milidetik buzzer berbunyi (default: 100ms)

**Perilaku:**
- SET BUZZER_PIN HIGH (buzzer active-low)
- Tunggu selama durasi
- SET BUZZER_PIN LOW

**Dipanggil oleh:** Deteksi kartu, error, notifikasi sukses

---

#### `void loadConfig()`

**Tujuan:** Memuat konfigurasi perangkat dari LittleFS `/config.json`.

**Alur:**
1. Mount filesystem LittleFS (format jika mount gagal)
2. Cek apakah `/config.json` ada
3. Jika ada, baca dan parse JSON
4. Populasi: `wifiNetworks[]`, `serverHostname`, `serverPort`, `deviceCode`, `pairingKey`
5. Jika tidak ada atau error, gunakan nilai default

**Dipanggil oleh:** `setup()`

---

#### `void saveConfig()`

**Tujuan:** Menyimpan konfigurasi perangkat saat ini ke LittleFS `/config.json`.

**Alur:**
1. Buat dokumen JSON dengan semua nilai konfigurasi
2. Buka `/config.json` untuk menulis
3. Serialisasi JSON ke file
4. Tutup file

**Dipanggil oleh:** `handleSave()`

---

#### `void handleRoot()`

**Tujuan:** Menyajikan halaman HTML portal konfigurasi.

**Alur:**
1. Muat template HTML dari PROGMEM
2. Ganti semua placeholder token dengan nilai saat ini
3. Kirim response HTML (200)

**Dipanggil oleh:** ESP8266WebServer saat klien meminta `/`

---

#### `void handleSave()`

**Tujuan:** Memproses submit form konfigurasi.

**Alur:**
1. Baca semua field form dari data POST
2. Update `wifiNetworks[]`, `serverHostname`, `serverPort`, `deviceCode`, `pairingKey`
3. Bersihkan hostname (hapus suffix `.local` jika ada)
4. Panggil `saveConfig()`
5. Kirim konfigurasi sync JSON ke ESP32-CAM via Serial: `"SYNC_JSON|{...}"`
6. Kirim response HTML sukses
7. Tunggu 2 detik
8. Reboot ESP8266 (`ESP.restart()`)

**Dipanggil oleh:** ESP8266WebServer saat klien POST ke `/save`

---

#### `void startConfigPortal()`

**Tujuan:** Memulai mode Access Point WiFi untuk konfigurasi perangkat.

**Alur:**
1. Set `inConfigMode = true`
2. Set mode WiFi ke AP
3. Mulai AP dengan SSID `"AttendTrack-Setup"`, password `"12345678"`
4. Tampilkan IP AP (`192.168.4.1`) di LCD
5. Daftarkan route web server (`/` dan `/save`)
6. Mulai web server
7. Masuk ke infinite loop menangani request klien

**Tampilan LCD:** "CONFIG MODE" / "192.168.4.1"

**Dipanggil oleh:** `setup()` saat tidak ada WiFi dikonfigurasi atau semua koneksi gagal

---

#### `bool connectWiFi()`

**Tujuan:** Mencoba terhubung ke jaringan WiFi yang dikonfigurasi.

**Alur:**
1. Set mode WiFi ke STA
2. Iterasi melalui `wifiNetworks[]`
3. Untuk setiap jaringan:
   - Tampilkan pesan koneksi di LCD
   - Panggil `WiFi.begin(ssid, password)`
   - Tunggu hingga 10 detik (20 × 500ms attempts)
   - Jika terhubung, return `true`
4. Jika semua jaringan gagal, return `false`

**Returns:** `true` jika terhubung, `false` jika gagal

**Dipanggil oleh:** `setup()`, `loop()` (reconnection)

---

#### `void resolveBackend()`

**Tujuan:** Menemukan server backend menggunakan mDNS (Bonjour).

**Alur:**
1. Inisialisasi mDNS responder
2. Query untuk service `_attendtrack._tcp`
3. Coba hingga 5 kali (interval 1 detik)
4. Jika ditemukan: simpan `resolvedServerUrl = "http://" + IP + ":" + port`
5. Tampilkan hasil di LCD
6. Jika tidak ditemukan: tampilkan "SERVER OFFLINE"

**Dipanggil oleh:** `setup()`, `loop()` (re-discovery setelah reconnect)

---

#### `void showIdle()`

**Tujuan:** Menampilkan pesan idle/siap di LCD.

**Tampilan LCD:**
- Baris 1: "Ready to Scan"
- Baris 2: "Tempel Kartu..."

**Dipanggil oleh:** `setup()`, `loop()` (setelah scan selesai)

---

#### `String readUID()`

**Tujuan:** Membaca UID kartu RFID dari MFRC522.

**Returns:** String hex UID (contoh: "A1B2C3D4"), huruf besar

**Alur:**
1. Iterasi melalui `mfrc522.uid.uidByte[]`
2. Konversi setiap byte ke string hex 2 karakter
3. Gabungkan dan konversi ke huruf besar

**Dipanggil oleh:** `loop()` saat kartu terdeteksi

---

#### `void setup()`

**Tujuan:** Inisialisasi semua hardware dan koneksi jaringan.

**Alur:**
1. Inisialisasi Serial (115200 baud) dan linkSerial (9600 baud)
2. Inisialisasi I2C dan LCD
3. Nyalakan LCD backlight
4. Konfigurasi pin buzzer sebagai OUTPUT (LOW = mati)
5. Tampilkan splash screen "AttendTrack V2"
6. Muat konfigurasi dari LittleFS
7. Cek apakah ada WiFi dikonfigurasi
8. Jika tidak ada WiFi atau koneksi gagal → mulai config portal
9. Resolve backend via mDNS
10. Inisialisasi bus SPI dan MFRC522
11. Tampilkan idle screen

---

#### `void loop()`

**Tujuan:** Loop utama program — memeriksa RFID, berkomunikasi dengan backend.

**Alur:**
```
1. Skip jika inConfigMode
2. Cek scan cooldown (3000ms)
3. Cek status WiFi → reconnect jika perlu
4. Cek kartu RFID baru
   a. Jika tidak ada kartu → return
   b. Jika kartu terdeteksi:
5. Baca UID
6. Beep (100ms)
7. Update lastScanAt
8. Tampilkan "Memproses..." di LCD
9. Resolve backend jika belum di-resolve
10. Jika server offline → beep error, tunggu, show idle
11. POST ke /api/v1/attendance/check-rfid
    a. Jika 200 OK:
       - Parse response
       - Jika "registered": false:
         - Jika "action": "REGISTER_CAPTURE" → kirim "REGISTER_CAPTURE|UID" ke ESP32-CAM
         - Lainnya → tampilkan "RFID TIDAK TERDAFTAR"
       - Jika "registered": true → lanjutkan
    b. Jika tidak 200 → tampilkan "Koneksi Error"
12. Kirim "CAPTURE|UID" ke ESP32-CAM via Serial
13. POST ke /api/v1/attendance/rfid (timeout 25s)
    a. 200 → "Wajah Cocok! Verified" + beep panjang
    b. 400/401 → "Wajah Tdk Cocok" + beep error
    c. Lainnya → "Koneksi Timeout" + beep error
14. Tunggu 3 detik
15. Halt kartu RFID
16. Tampilkan idle screen
```

## 6.3 Protokol Serial (ke ESP32-CAM)

Perintah dikirim via SoftwareSerial (9600 baud):

| Perintah | Format | Tujuan |
|----------|--------|--------|
| CAPTURE | `CAPTURE|{UID}\n` | Ambil foto wajah untuk absensi |
| REGISTER_CAPTURE | `REGISTER_CAPTURE|{UID}\n` | Ambil foto wajah untuk registrasi |
| SYNC_JSON | `SYNC_JSON|{json}\n` | Sinkronisasi konfigurasi |

## 6.4 Endpoint HTTP yang Dipanggil

| Endpoint | Method | Tujuan | Timeout |
|----------|--------|--------|---------|
| `/api/v1/attendance/check-rfid` | POST | Cek apakah RFID terdaftar | 5s |
| `/api/v1/attendance/rfid` | POST | Verifikasi RFID + wajah | 25s |

## 6.5 Penanganan Error & Recovery

- **WiFi Disconnect:** `loop()` memeriksa `WiFi.status()` dan reconnect
- **Semua WiFi Gagal:** Fallback ke config portal
- **Server Tidak Ditemukan:** Tampilkan "SERVER OFFLINE", retry pada scan berikutnya
- **HTTP Timeout:** `check-rfid` = 5s, `rfid` = 25s (lebih lama karena menunggu verifikasi wajah)
- **Scan Cooldown:** 3000ms antar scan untuk mencegah spam RFID
- **Buzzer Feedback:** Pola beep berbeda untuk sukses (500ms) vs error (3 × 100ms)
